## 📌 О проекте

Этот проект создан в учебных целях для изучения автоматизации UI-тестирования с использованием Selenium WebDriver (+pytest) на Python.

На текущем этапе полностью покрыты тестами следующие разделы сайта demoqa.com:
- **Elements**
- **Forms**
- **Alerts, Frame & Windows**
- **Widgets**
- **Interactions**

**Общее количество тестов:** 77

**Allure‑отчёты** интегрированы для наглядного представления результатов.  
Почти все тесты защищены от парадокса пестицида: используются генерация случайных данных, выбор случайного количества элементов, рандомизация порядка действий и т.п., что позволяет выявлять скрытые ошибки и избегать «залипания» тестов на одних и тех же сценариях.

**GitHub Actions** настроен для ручного запуска тестов и автоматической публикации Allure‑отчёта на **GitHub Pages**.  
[Посмотреть свежий отчёт Allure](https://bogdanxqa.github.io/demoqa_selenium_automation/)

Проект можно бесконечно улучшать и для меня он еще не является завершённым, однако уже демонстрирует мои навыки построения фреймворка с нуля, работы со сложными взаимодействиями (drag-and-drop, iframes, вкладки, алерты, верификация больших форм, селекты, модальные окна и т. д.), применения паттерна Page Object, кастомных ожиданий, формирования отчётов и настройки GitHub Actions.

## 🧱 Что уже автоматизировано:

### Раздел Elements

| Страница | Что проверяется |
|----------|----------------|
| Text Box | Заполнение формы, отправка, проверка отображения введённых данных |
| Check Box | Раскрытие всего дерева, выбор случайного чекбокса, проверка результата |
| Radio Button | Выбор каждой радио-кнопки (Yes, Impressive) и проверка сообщения |
| Web Tables | CRUD-операции: добавление, редактирование, удаление, поиск, пагинация |
| Buttons | Двойной клик, правый клик, обычный клик с проверкой сообщения |
| Links | Клик по всем ссылкам, проверка текста ответа (200, 201, 400 и др.), requests |
| Upload and Download | Генерация файла, загрузка и проверка успешности |
| Dynamic Properties | Ожидание изменений элементов (цвет, доступность, появление, работа с динамическим id) |

### Раздел Forms

| Страница | Что проверяется |
|----------|----------------|
| Practice Form | Полное заполнение формы (генерация случайных данных: имя, фамилия, email, телефон, адрес, выбор радиокнопки, чекбоксов, предметов, файла, state/city). После отправки – сравнение ожидаемых данных с отображёнными в модальном окне. Дополнительно: выявлен баг – кнопка Close не закрывает окно (падающий тест), закрытие работает только кликом на overlay. |

### Раздел Alerts, Frame & Windows

| Страница | Что проверяется |
|----------|----------------|
| Browser Windows | Открытие новой вкладки, нового окна, окна с сообщением; переключение и проверка текста |
| Alerts | Обычный alert, alert с таймером, confirm box (с параметризацией), prompt box |
| Frames | Переключение во фреймы, проверка текста внутри |
| Nested Frames | Переключение в родительский и дочерний фреймы, проверка текста |
| Modal Dialogs | Открытие и закрытие, работа модальных окон (Small и Large) |

### Раздел Widgets

| Страница | Что проверяется |
|----------|----------------|
| Accordian | Раскрытие каждой секции, проверка заголовка и что текст контента не пустой |
| Auto Complete | Ввод случайных значений, проверка, что выбранные значения отображаются корректно (сравнение с нормализованным списком) |
| Date Picker | Выбор случайной даты (и даты+времени), проверка соответствия отображаемого значения. Работа с Select. |
| Slider | Перемещение ползунка на случайное расстояние, проверка изменения значения. Метод drag and drop by offset |
| Progress Bar | Запуск прогресс-бара, случайная остановка, проверка изменения; также полный цикл сброса по кнопке Reset |
| Tabs | Переключение между вкладками (What, Origin, Use, More), проверка отображаемого контента |
| Tool Tips | Наведение на кнопку, поле ввода, ссылки – проверка всплывающих подсказок |
| Menu | Наведение на пункты меню (главное, подменю), сбор всех заголовков |
| Select Menu | Выбор из стандартного select, кастомных выпадающих списков (Select Value, Select One, Old Style Select, Multi‑select). Сравнение выбранного и отображённого значений |

### Раздел Interactions

| Страница | Что проверяется |
|----------|----------------|
| Sortable | Перетаскивание элементов в списке и сетке (List / Grid) с проверкой порядка после перемещения |
| Selectable | Выбор случайных неактивных элементов в списке и сетке; проверка, что после клика элемент становится активным |
| Resizable | Изменение размеров блока с ограничениями (min/max) и без ограничений; проверка изменения стиля (width/height) |
| Droppable | Drag-and-drop с проверкой реакции зон (Accept, Prevent Propagation, Revert Draggable, Greedy/Not Greedy). Сравнение цвета и текста до/после, а также поведения при перетаскивании на outer/inner зоны |
| Dragabble | Перетаскивание простого блока, ограничение по оси X/Y, движение внутри родительского контейнера (проверка границ), изменение положения по смещению |

## 🚀 Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/bogdanxQA/FINAL_PROJECT_SELENIUM.git
cd FINAL_PROJECT_SELENIUM
```

---

### 2. Создание виртуального окружения

```bash
python -m venv venv
venv\Scripts\activate      # Windows
source venv/bin/activate   # Linux/Mac
```

---

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

---

### 4. Запуск всех тестов

```bash
pytest -v -s
```

---

### 5. Запуск конкретного теста

```bash
pytest -k "TestWebTablesPage" -v -s
pytest tests/test_elements.py::TestElements::TestWebTablesPage::test_add_new_data -v -s
```
